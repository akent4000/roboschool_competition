Память объектов (3 уровня)

  • Depth-confirmed — мгновенное сохранение с высокой точностью позиции
  • Sight accumulator — ≥3 наблюдений в радиусе 2м, усреднение позиции
  • Stale eviction — depth-подтверждение нового объекта вытесняет старую ошибку в радиусе 1.5м

Масштабируемость пайплайна

  • Разные seed = разные раскладки комнат без ручной работы
  • Добавление нового класса объекта — один запуск скрипта сбора
  • Негативные примеры собираются автоматически
  • Интерактивный просмотрщик для контроля качества

SLAM и навигация

Полный стек с нуля: от сырого depth до плавных скоростных команд

Карта занятости

  • 800×800 ячеек, 5 см/ячейка (40×40м)
  • Log-odds модель: L_OCC=0.85 L_FREE=-0.4
  • DDA raycasting по depth-лучам
  • Vectorized depth projection (numpy)
  • Inflated obstacle mask (OpenCV dilation)

Frontier exploration

  • Граница свободного и неизвестного пространства
  • Кластеризация через flood-fill
  • Стратегия least_explored: доля неизвестного + размер − расстояние
  • Исключение зон в 1.5м от посещённых объектов

A* планировщик

  • 8-направленный поиск на inflated-карте
  • Штраф за повороты λ=0.3 → плавные пути
  • Line-of-sight smoothing + densification 0.2м
  • Лимит 200K итераций для предсказуемого времени

Pure Pursuit контроллер

  • Lookahead = 0.8м, адаптивная скорость по кривизне
  • Boost до 1.2 м/с на прямых (ошибка < 8°)
  • Obstacle avoidance: стоп 0.55м, замедление 0.55–1.0м
  • Low-pass фильтр (α=0.3) + dead zone ±4°

Anti-stuck (3 уровня)

Sticky backing 1.2s Hard turn 0.6s Random walk 2s

Масштабируемость

Решение готово к расширению на новые объекты, пространства и железо

Объекты
Больше классов
Менеджер очереди и память работают с произвольным числом классов. Добавление нового = один запуск collect_dataset.py + дообучение YOLO. Без ручной разметки.
Пространство
Большие комнаты
Карта 40×40м параметризована. Разрешение меняется одной строкой. A* ограничен 200K итераций — при длинном пути робот доедет до промежуточной точки и перепланирует.
Железо
Реальный робот
Абстрактный API: set_speed, get_state, get_camera. Переход на реальное железо — только реализация интерфейса. Навигация не меняется.
Сенсоры
Другие датчики
SLAM принимает depth как массив. Замена камеры на LiDAR затрагивает только модуль проекции. Остальной стек не меняется.
Модульность
Независимая замена
Каждый компонент — детектор, SLAM, контроллер — заменяем независимо благодаря чётким интерфейсам. Можно подставить RTAB-Map или DWA без переписывания остального.
